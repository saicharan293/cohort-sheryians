This page explains commonly used HTML tags using only pure HTML. No CSS is used. The beauty comes from proper structure and semantic tags.

1. Document Structure Tags

<!DOCTYPE html>

Declares the document type and version of HTML. It tells the browser that this document follows HTML5 standards.

<html>

The root element of an HTML page. All other elements are placed inside this tag.

<head>

Contains metadata like title, character encoding, viewport settings, and links to external resources.

<body>

Contains all the visible content of the webpage such as text, images, forms, and buttons.

2. Head Tags (Inside <head>)

Head tags provide information about the webpage to browsers and search engines. These tags do not appear directly on the page.

<meta charset="UTF-8">

Specifies the character encoding for the document. UTF-8 supports most languages, symbols, and emojis.

<meta name="viewport">

Controls how the webpage is displayed on different screen sizes. It is essential for responsive and mobile-friendly design.

<title>

Defines the title of the webpage. It appears in the browser tab, bookmarks, and search engine results.

<meta name="description">

Provides a short description of the webpage. Search engines may display this in search results.

<link>

Used to connect external resources like CSS files, icons (favicon), or fonts to the HTML document.

<style>

Allows writing internal CSS inside the HTML document. Useful for small styling rules.

<script>

Used to add JavaScript code or external JavaScript files. Scripts in the head often use the defer attribute.

<base>

Defines a base URL for all relative links in the document.

3. Text & Heading Tags

<h1> to <h6>

Used for headings. <h1> is the most important heading and <h6> is the least important.

<p>

Used to define a paragraph of text.

<br>

Inserts a line break without starting a new paragraph.

<hr>

Creates a horizontal line, often used to separate sections.

4. Form Tags

Form Elements
<form>

Used to collect user input and send data to a server.

<input>

Used to take user input. It supports many types like text, email, date, radio, checkbox, and more.

<label>

Provides a label for input fields and improves accessibility.

<button>

Represents a clickable button used to submit forms or perform actions.

5. Interactive & Semantic Tags

<details> & <summary>

Creates expandable and collapsible sections. Useful for FAQs and extra information.

<fieldset> & <legend>

Groups related form elements together with a caption.

<meta>

Provides metadata such as character encoding, viewport settings, and SEO-related information.

6. Why Semantic HTML Matters

  • Improves readability
  • Enhances accessibility
  • Helps search engines understand content
  • Creates clean and maintainable code
