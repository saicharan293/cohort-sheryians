
## CLIENT - SERVER (Model) ARCHETECTURE
----------------------------------------

# CLIENT :
----------
1. Client is a device or program (like browser or an app) that requests data

# SERVER =
-----------
1. Server is a computer system that stores, processes, delivers (responds) the data requested by client 

## HTTP Request - Response Cycle:
---------------------------------

1. User hits an url (sheryians), browser sends HTTP requests to the server (request)
2. Server processes the data (process)
3. Server returns/send the response back (response)
4. browser renders the data. (renders)

## Website working
-----------------

1. When a link is typed in browser
2. Browser sends the HTTP request to ISP , -> Dns Lookup
3. ISP routes the request to that domain server
4. Server processes the HTTP request and sends response
5. Browser renders the response and display the website
